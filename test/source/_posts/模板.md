## 4. Hexo 常见 Markdown 能力

### 4.1 任务清单

- [x] 完成需求梳理
- [x] 完成技术评审
- [ ] 完成联调回归
- [ ] 完成生产发布

### 4.2 引用与强调

> 对外承诺功能必须有可验证验收项，对内技术方案必须有可回滚路径。

### 4.3 代码块（含语言高亮）

```bash
npm run clean
npm run build
npm run server -- --open --watch
```

```js
const milestones = [
	{ name: '需求冻结', date: '2026-04-25' },
	{ name: '联调完成', date: '2026-05-03' },
	{ name: '灰度发布', date: '2026-05-08' }
]

console.table(milestones)
```

### 4.4 数学公式

若版本收益记作 $R$，开发成本记作 $C$，可粗略表示为：

$$
ROI = \frac{R - C}{C}
$$

## 5. 验收标准

1. 导入失败记录支持二次下载。
2. 关键操作均写入审计日志且带操作者信息。
3. 首页摘要显示正常，详情页封面、标签插件渲染正常。

## 6. 回滚预案

1. 保留旧接口兼容 1 个小版本。
2. 提供数据库结构回滚脚本。
3. 发布后 30 分钟内重点观察错误率与慢查询。
